        # 基础知识

        刚开始学习 vue 时也是直接看文档， 其中遇到一些不懂的知识点会自动跳过，而往后也没有回去补充这部分的知识，此文就是回来填之前跳过的坑。

        # 编译器+运行时 vs 运行时

        在 vue 官网手册安装篇中提到 vue 其实是有两个版本，一个是完成版（包括编译器+运行时）和仅仅是运行时。

        编译器是将template字符串转化为 javascript 的。

        // 需要编译器
new Vue({
  template: "<div>{{ hi }}</div>",
});

// 不需要编译器
new Vue({
  render(h) {
    return h("div", this.hi);
  },
});

        # render 函数和 JSX

        目前可以清楚的知道，在 vue 中的模板是要编译成 render 函数，执行 render 函数则会得到虚拟 dom

        render 函数长这样

        createElement(
  "anchored-heading",
  {
    props: {
      level: 1,
    },
  },
  [createElement("span", "Hello"), " world!"]
);

        <anchored-heading :level="1"> <span>Hello</span> world! </anchored-heading>

        TIP

        createElement函数返回 vnode，其参数包含 3 大部分，
        createElement(

        • "一个 HTML 标签名、组件选项对象，或者
        • resolve 了上述任何一种的一个 async 函数。必填项",
        • "一个与模板中 attribute 对应的数据对象。可选。子级虚拟节点 (VNodes)，由 createElement() 构建而成"
          )

        按照惯例createElement会被简写为 h

        然后实际发现直接手写 render 函数对于一些结构复杂的组件并不友好，也没有必要（程序员为何为难程序员），因此 jsx 出现了 (实质上也是 react 的东西)

        <AnchoredHeading level={1}>
  <span>Hello</span> world!
</AnchoredHeading>

new Vue({
  el: "#demo",
  render: function(h) {
    return (
      <AnchoredHeading level={1}>
        <span>Hello</span> world!
      </AnchoredHeading>
    );
  },
});

        TIP

        查看源码你会发现模板编译出来的 render 实际就是调用 _c 方法，用户自定义的 render实际就是 vm.$createElement 方法。
        这两个方法支持相同的参数，实际调用的都是 createElement 方法。
        因此使用模板编译后还需要调用render

        component和extend

        component：注册或获取全局组件。注册还会自动使用给定的 id 设置组件的名称

        extend：返回一个继承自父类的构造函数。参数是一个包含组件选项的对象。

        component 注册/获取全局组件

        // 注册组件，传入一个扩展过的构造器
Vue.component(
  "my-component",
  Vue.extend({
    /* ... */
  })
);

// 注册组件，传入一个选项对象 (自动调用 Vue.extend)
Vue.component("my-component", {
  /* ... */
});

// 获取注册的组件 (始终返回构造器)
var MyComponent = Vue.component("my-component");

        template

        一个字符串模板作为 Vue 实例的标识使用。模板将会替换挂载的元素。挂载元素的内容都将被忽略，除非模板的内容有分发插槽。

        如果值以 # 开始，则它将被用作选择符，并使用匹配元素的 innerHTML 作为模板。常用的技巧是用 <script type="x-template"> 包含模板。

        TIP

        ⚠️ 在单文件组件中，template标签 > render选项 > template选项

        小结 👈🏻
        虽然已经写到小结，但仍不太清楚 vue 各个 api 作用，简单理解下：
        👉🏻vue 具有 2 种方式构建页面，一种是官方推荐的 vue 单模版组件依靠编译生成 render 函数，另一种是直接写 render 函数，若直接写 render 函数太痛苦，还可以写 jsx 语法，依靠插件转译成 render 函数。
        👉🏻extend api，该 api 提供灵活的方式创建组件，可以结合component创建全局组件，亦可以直接挂在到某个元素，更加可以获取$el 属性直接 append 到 dom 树。
        👉🏻 compile将一个模版字符串编译成 render 函数，貌似比较少用。
