      # async/await

      # 事件循环

      # 结论

      • async 函数是什么？一句话，它就是 Generator 函数的语法糖。
      • 当 async 函数里面没有 await 语句时，代码块像同步代码那样执行,当 await 一个普通值也会包装成 Promise.resolve 相当于同步代码执行。
      • await一个异步 Promise 时，会立即执行其中的同步代码，随后如果有异步代码会执行其async外得同步代码，阻塞async内部代码，外部外吗执行完后，执行异步 resolve
      • 当 async 函数里有使用 await 时，在第一个await中等待该 promise（即使不是也会被封装成 promise）执行完成返回，在此行之后的所有代码都会打包成异步微事件推去队列。

        TIP

        补充一点：在上下文环境中理解，本层上下文环境 fn 是 async 时，遇到 await 就会阻塞本函数代码，若是异步就先执行函数外部同步代码，其余代码带包推进微事件队列

      # 代码

      async function forLoop() {
  Promise.resolve(console.log("gg"))

async function forLoop() {
  await new Promise((resolve) => {
    resolve()
  }).then(() => console.log(1))
  await console.log("xx")
}
forLoop()
console.log("yy")
//awai等待primise resolve 放进微消息队列，后边阻塞，此时主同步代码输出 yy、，继而执行微事件输出1、xx

async function forLoop() {
  await new Promise((resolve) => {
    setTimeout(resolve, 1000)
  }).then(() => console.log(1))
  await console.log("xx")
}

      # 分析

      TIP

      对于调用栈、异步队列、宏任务微任务等可以先翻看知识点中的理论知识

      代码由上往下执行

      1. 第一行，同步代码输出 1
      2. 第二行，遇到函数定义跳过
      3. 第七行，遇到 setTimeout 异步宏任务，添加进宏任务队列
      4. 第十行，调用 fn 函数发现里面可能存在异步方法且有可能需要阻塞
      5. 第三行，输出 2
      6. 第四行，遇到 await 异步表达式，虽然不是 promise 但会被包装成 promise 且阻塞至输出 3
      7. 第五行，由于上面是 await 语句，此后所有代码会被推进微任务队列，即输出 4 被推进微任务队列
      8. 第十一行，返回 promise 异步函数，执行其中同步代码输出 6，同时输出 7 被推进微任务队列
      9. 第十七行，执行同步代码输出 8， 至此同步代码及阻塞输出顺序为，1、2、3、6、8
      10. 执行微任务，输出 4，7
      11. 执行宏任务，输出 5

      完整输出为 1、2、3、6、8、4、7、5

      # 结论

      当 async 函数里有使用 await 时，会await等待该 promise（即使不是也会被封装成 promise）执行完成返回，在此行之后的所有代码都会打包成异步微事件推去队列。

      # 分析

      1. 全局作用域，执行全局上下文，完了后，进入 forLoop 函数上下文，
      2. forLoop 入栈，执行上下文
      3. await promise1,阻塞 forLoop 后边代码直到 resolve 或 reject 后才执行，执行 promise1 同步代码，
      4. 发现 setTimeout1 是异步宏事件，挂起等待执行，没有其他同步代码，此时回调到主线程，执行主线程剩余代码，执行输出 yy 5. 此时等待 1s
      5. 主线程没有代码，setTimeout1 倒计时结束，回调推进异步事件
      6. 主线程执行异步队列，执行 resolve1，输出 1，此后所有代码打包进微事件队列，
      7. 主线程执行异步微事件队列，把微事件队列代码放入主线程执行
      8. 发现 await2 也是个异步函数，同样执行其同步代码，挂起 setTimeout2，把往后代码同样打包成异步微任务推进队列，
      9. 执行同步代码发现没有，阻塞代码直到 setTimeout2 推进事件队列
      10. 此时等待 1s
      11. 主线程执行回调，输出 2，
      12. 同样，因为已经 resolve，await2 执行完毕，同步代码没有，执行微事件队列
      13. 把微事件队列代码放进执行栈执行，
      14. 发现 await3 也是个异步函数，同样执行其同步代码，挂起 setTimeout3，把往后代码同样打包成异步微任务推进队列，
      15. 此时等待 1s
      16. 执行 setTimeout3 回调，输出 3，此时没有同步代码，直接执行微任务，输出 xx
